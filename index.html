About this repo

What is Yii2?

This is best explained by the folks at Yii themselves:

Yii is a free, open-source Web application development framework written in PHP5 that promotes clean, DRY design and encourages rapid development. It works to streamline your application development and helps to ensure an extremely efficient, extensible, and maintainable end product.

To learn more, head over to the official Yii website

What is Materialize?

Materialize is a bunch of CSS components based on Google's Material Design.

Created and designed by Google, Material Design is a design language that combines the classic principles of successful design along with innovation and technology. Google's goal is to develop a system of design that allows for a unified user experience across all their products on any platform.

For a full overview of all Materialize features visit the project website.

Wiring up

As a de facto standard Yii2 is shipped with a Bootstrap extension allowing for out of the box styled views in your application.

This repo provides an alternative to Bootstrap by implementing Materialize into Yii2.

Bootstrap is a really fine piece of CSS (especially with version 4 on the horizon). The intention of this repo is to show you another CSS framework and maybe to encourage you to use it your next Yii2 application.

Getting started

Get Composer

The preferred way of installation is through Composer. If you don't have Composer you can get it here: https://getcomposer.org/

You also should install the Composer Asset Plugin to handle NPM and Bower assets: 

                    
                        $ composer global require "fxp/composer-asset-plugin:~1.4"

Configure your app dependencies

To install the package add the following to the require section of your composer.json: 

                    
                        "require": {
                            "macgyer/yii2-materializecss": "*"
                        },

Load Materialize

To load the Materialize CSS files integrate the MaterializeAsset.php into your app. One way to achieve this is to register the asset in the main layout: 

                    
                        // @app/views/layouts/main.php

                        \macgyer\yii2materializecss\assets\MaterializeAsset::register($this);

                        // further code

Another way is to define a dependency in your app wide AppAsset.php 

                    
                        // @app/assets/AppAsset.php

                        public $depends = [
                            'macgyer\yii2materializecss\assets\MaterializeAsset',

                            // more dependencies
                        ];

Gii Support

If you are creating your CRUD controller and view files using Gii you can get materialized view files by integrating the adapted Gii templates. 

                    
                        // @app/config/main-local.php

                        $config['modules']['gii'] = [
                            'class' => 'yii\gii\Module',
                            'allowedIPs' => ['127.0.0.1', '::1', '192.168.0.*', '192.168.178.20'],
                            'generators' => [
                                'crud' => [
                                    'class' => 'yii\gii\generators\crud\Generator',
                                    'templates' => [ // setting materializecss templates
                                        'materializecss' => '@vendor/macgyer/yii2-materializecss/src/gii-templates/generators/crud/materializecss',
                                    ]
                                ]
                            ],
                        ];

You can copy those templates to any location you wish for further customisation. Make sure you adapt the path accordingly in your config.
